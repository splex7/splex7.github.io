Github에서 Hexo 시작하기

Posted on Aug 31 2019

과거에 블로그라하면 네이버블로그나 티스토리를 떠올렸지만,
SEO에 대한 이해가 높아지고 구글애드센스와같은 광고수익모델이 보편화되면서
자잘한 제약이 많은 블로그 플랫폼을 떠나는 사람들이 많아졌다.

##정적사이트생성기 SSG
워드프레스나 블로거와 같은 툴도 큰 인기를 끌었지만
깃허브나 깃랩이 제공하는 무료 (정확히는 한달에 100GB에 달하는 soft limit)
페이지 호스팅과 정적 사이트 생성기 Static Site Generator (SSG)를 이용하면
각종 테마와 플러그인을 직접 설치해가며 나에게 꼭 맞는 블로그를 구축하면서도
플랫폼의 운영사의 방침에 얽매이지 않는 블로그를 구축할 수 있다.

향후 다시 언급하겠지만, 깃랩 페이지에서의 경험은 별로 좋지 않았기때문에
깃허브 페이지를 추천하고싶다.

2019년 현재 인기있는 Static Site Generator (SSG)
가장 널리 사용되는 루비기반의 Jekyll,
NodeJS 기반의 Hexo, Gatsby 등이 있고
더 알아보지는 못했지만 Hugo, Middleman, Harp, Brunch 같은 것들도 있다.
차후 이러한 SSG들의 특징을 조사해서 포스팅해보겠다.

##Hexo with Github Pages
우선 나는 Gitlab Pages 서비스를 통해 HEXO 를 호스팅해보았는데,
Netlify에서 Gatsby를 호스팅할때 너무나 모든게 마법처럼 됐던 경험과는
끔직하게 다른 경험이었다. Repo명명법부터 커스텀도메인, CSS path문제가 계속해서
발목을 잡았고, 제대로 사이트를 띄우기까지 너무나 오랜시간이 걸렸다.

이게 단지 Gitlab Pages의 Hexo와의 궁합문제인지는 모르겠으나
Github Pages에서 Hexo를 실행시켰을때 훨씬 쉽게 모든게 진행됐다.
